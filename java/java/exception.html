finally

return finally throw 冲突

  • finally优先级最高,只要进入try,软件没挂,都会执行
  • 方法中断覆盖 return和throw相续入栈,后发生者生效
  • try中throw后,跳到catch,catch中throw后,跳到finally
  • try和catch中的return执行完计算部分,并将其值保存入栈,跳到finally
  • finally内对已入栈的返回变量的值的修改不会对返回值产生影响
  • finally内无return,再执行try或catch中的return
  • finally中的return会覆盖try或catch中的return
