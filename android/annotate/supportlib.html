Android Support Annotations Library

配置

  • compile 'com.android.support:support-annotations:22.2.0'

Nullness Annotations

  • @Nullable 标注给定的参数或者返回值可以为null
  • @NonNull 标注给定的参数或者返回值不能为null

资源类型注解 @XxxRes int resId

  • @StringRes, @DrawableRes, @ColorRes, @InterpolatorRes ...
  • @AnyRes 标注一个未知的特殊类型的资源，但是它必须是一个资源类型
  • 使用多个注解来支持多个资源类型

IntDef/StringDef: 类型定义注解

线程注解

  • 标记的方法只能在指定的线程类型中被调用
  • 注解类本身 所有方法都有相同的线程要求
  • @UiThread
  • @MainThread
  • @WorkerThread
  • @BinderThread

RGB颜色整型

  • @ColorRes 颜色资源
  • @ColorInt 代表颜色的整数值

值约束

  • @Size()

    @Size(min=2)
    @Size(max=2)
    @Size(2)
    @Size(multiple=2) 倍数

  • @IntRange(from=0,to=255)
  • @FloatRange(from=0.0, to=1.0)

权限注解

  • @RequiresPermission(Manifest.permission.SET_WALLPAPER)

    方法的调用需要调用者有特定的权限

  • @RequiresPermission(anyOf = {......})

    至少需要权限集合中的一个

  • @RequiresPermission(allOf = {.....})

    同时需要多个权限

  • 对于intents的权限 
    
@RequiresPermission(android.Manifest.permission.BLUETOOTH)
public static final String ACTION_REQUEST_DISCOVERABLE =
        "android.bluetooth.adapter.action.REQUEST_DISCOVERABLE";
  • 对于content providers的权限 
    
@RequiresPermission.Read(@RequiresPermission(READ_HISTORY_BOOKMARKS))
@RequiresPermission.Write(@RequiresPermission(WRITE_HISTORY_BOOKMARKS))
public static final Uri BOOKMARKS_URI = Uri.parse("content://browser/bookmarks");

方法重写

  • @CallSuper

    父方法被标记时,要求重写方法调用父方法

返回值处理

  • @CheckResult

    调用者要使用方法返回值

  • @VisibleForTesting
  • @Keep 不会被混淆
