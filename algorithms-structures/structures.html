Data Structures

数据结构(Data Structure)

  • 计算机存储、组织数据的方式
  • 相互之间存在一种或多种特定关系的数据元素的集合
  • 数据(data):数据是指计算机能够接受和处理的一切对象
  • 数据元素(data element):数据元素是指在计算机中作为一个整体考虑和处理的对象, 是组成数据的基本单位
  • 数据对象(data object):数据对象是指性质相同的若干数据元素的集合, 是数据的子集。
  • 数据的逻辑结构(logic structure):数据元素之间的逻辑关系
  • 数据的存储结构(storage structure)或物理结构(physical structure)
  • 数据的操作(operation):数据之间所要进行的运算

数据的的逻辑结构:线性,非线性,集合

  • 线性结构:线性表、栈和队列、 串、数组、广义表
  • 非线性结构:树、二叉树、有向图、无向图以及集合类型

数据的存储结构:顺序,链式,索引,散列

  • 顺序(Sequential)存储方法:把每个结点的数据, 按照某种顺序存放在一段连续的存储单元中
  • 链式(Linked)存储方法:把每个结点的数据, 零散地存放在一些存储单元中
  • 索引(Index)存储方法:用结点的索引号i来确定结点的存储单元地址, 把每一个结点的数据按一定规律顺序或链式存放在存储单元中
  • 散列(Hash)存储方法:把每个结点的数据通过一个预设的散列函数, 来决定该结点的存储单元。

四种基本数据结构：

  • 集合：数据元素间“同属一个集合”。
  • 线性结构：一个对一个，如线性表，栈，队列。
  • 树形结构：一个对多个
  • 图状结构：多个对多个

常见的数据结构

  • 线性表(Linear List)
  • 栈(Stack)与队列(Queue)
  • 串
  • 数组(Array) 广义表
  • 树(Tree)
  • 图(Graph)
  • 堆(Heap)
  • 散列表(Hash)

数组Array

  • 由相同类型的元素的集合所组成的数据结构，分配一块连续的内存来存储,利用元素的索引计算出该元素对应的储存地址。

堆栈Stack

  • 串列形式的数据结构,只允许在链接串列的一端（堆叠顶端top）进行加入push和输出pop

队列Queue

  • 通常用链表或者数组来实现,只允许在后端rear插入，在前端front删除
  • 单链队列使用链表作为基本数据结果，所以不存在伪溢出的问题，队列长度也没有限制。但插入和读取的时间代价较高
  • 循环队列可以更简单防止伪溢出的发生，但队列大小是固定的。

链表Linked List

  • 线性表,在每一个节点里存到下一个节点的指针
  • 单向链表,在每个节点保存数据和到下一个节点的地址，在最后一个节点保存一个特殊的结束标记，另外在一个固定的位置保存指向第一个节点的指针
  • 双向链表,不仅有指向后一个节点的指针，还有指向前一个节点的指针
  • 循环链表,首节点和末节点被连接在一起
  • 块状链表,链表储存由数据组成的顺序表,每个节点就是顺序表，可以被叫做一个块

树Tree

  • 具有树状结构性质的数据集合

散列表Hash

  • 根据键而直接访问在内存存储位置的数据结构。通过计算一个关于键值的函数，将所需查询的数据映射到表中一个位置来访问记录
